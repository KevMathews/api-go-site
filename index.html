Introduction

Welcome to API-Go

The dev package designed to help jump start your React website!



DOWNLOAD HERE

Or get API-Go from Joshua or Kevin's GitHub account.

API-Go was created for one simple reason, to ease the use of APIs!
Anyone from an aspiring student, to a senior level dev can benefit from this package.

No more squandering precious dev time studying
up on how to interact with a particular API.

With API-Go that information, with examples, will be at
your fingertips for easy reference!

So, how exactly does API-Go save you time?


We accomplish this in 2 simple ways:

  • By presenting you a simplified 'cheat-sheet' of the endpoints and parameters for the API of your choice.

  • By providing examples of fetch calls, and showing you the data in json format that is returned.

Essentially, we save you time by packaging together the information needed to get
started with some commonly used Free APIs, and present the info to you in an
easy to read and access format.







Usage

To get started you can download API-Go from either GitHub or via our website.
Alternatively, you can also clone the directory from Github.
- We will be deploying soon via NPM -

Once the files are located on your computer, go into the folder and run 'npm i -g'
in the terminal to install our package globally on your computer.
Take note that some installations might require sudo or administration access

Once done, launch API-Go from the directory you want to create your project in.
*Make sure the directory you launch API-Go in ISN'T a git hub repo already *

Once inside the directory of choice, type api-go from your terminal command line

You will be presented with several questions while navigating on the command line

First you will be asked to choose which API you want to work with

Next, you need to enter an API key if you have one, and name your project
so we can properly create the directory structure for you.

After that is all completed, give the process up to a minute or so to download and
install the packages and node modules needed to get you up and running.

Once completed cd into the newly created directory that will contain all of the dynamically
created files for you and type 'npm start' in a terminal window to launch.


What you will have is essentially a 'cheat sheet' at your fingertips to be able to reference
query params and have example fetches to help jump start your development.

News API's everything endpoint Query Parameters:


PARAMETERS DESCRIPTION
apiKey Your API key. Alternatively you can provide this via the X-Api-Key HTTP header.
q Keywords or phrases to search for in the article title and body.
sources A comma-seperated string of identifiers (maximum 20) for the news sources or blogs you want headlines from.
domains A comma-seperated string of domains (mashable.com, techcrunch.com, engadget.com) to restrict the search to.
from A date and optional time for the oldest article allowed. This should be in ISO 8601 format (e.g. 2021-04-03 or 2021-04-03T15:46:09)
to A date and optional time for the newest article allowed. This should be in ISO 8601 format (e.g. 2021-04-03 or 2021-04-03T15:46:09)
language The 2-letter ISO-639-1 code of the language you want to get headlines for. Possible options: ar de en es fr he it nl no pt ru se ud zh

Fetch

As a Junior Dev, Fetch calls can be a tricky to setup, and very time consuming to figure out.

The example below is a React Hooks Fetch to the Nasa Photo of the Day API.
Overall a very simple fetch, as it's pretty straightforward, and has just 1 endpoint.

Let's walk you through the example above to hopefully demystify it some


We create photoData as a variable and setPhotoData as the function to update the variable's value.
React's useState hook is used to set the initial state of photoData to an empty object.
The async function fetchPhoto helps avoid asyncronous issues that can occur when completing
fetch requests. We use setPhotoData to update the photoData variable with the API's response.


Instead of using a lifecycle method, we call setData inside the useEffect React hook. The empty
array on Line 14 may look confusing at first, but let’s refer back to the React Hooks Documentation!

“If you want to run an effect and clean it up only once (on mount and unmount), you can pass an
empty array ([]) as a second argument. This tells React that your effect doesn’t depend on any values
from props or state, so it never needs to re-run. This isn’t handled as a special case — it follows
directly from how the dependencies array always works.

We pass the empty array into useEffect to ensure that the effect runs once and to prevent the
component from perpetually re-rendering. The function component’s return statement displays
attributes by calling the format photoData.url or photoData.title as example.

Both components now fetch NASA’s API successfully and render the image, title,
date, and description as depicted in the image below.
